### 2.2 关键架构原则

1. **精算中台是纯计算层** — 无状态、无 IO、无数据库读写。函数式接口: `input → compute → output`

### 3.1 极值理论引擎 (evt_engine.py)

**问题**: 当前用正态分布做 VaR，而 A 股收益呈现显著的厚尾特征（峰度通常 > 5）。

**精算方案**: Generalized Pareto Distribution (GPD) + Peaks-Over-Threshold (POT) 方法。

**数学基础**:

```
Pickands-Balkema-de Haan 定理:
对于充分高的阈值 u，超额损失 Y = X - u | X > u 的分布收敛于 GPD:

G_{ξ,β}(y) = {
  1 - (1 + ξy/β)^{-1/ξ},  ξ ≠ 0
  1 - exp(-y/β),          ξ = 0
}

其中 ξ = 形状参数 (尾指数), β = 尺度参数

VaR_α (EVT):
VaR_α = u + (β/ξ) * [(n/N_u * (1-α))^{-ξ} - 1],  ξ ≠ 0
VaR_α = u + β * ln(n/N_u * (1-α)),               ξ = 0

其中 n = 总观测数, N_u = 超过阈值的观测数

Expected Shortfall (EVT):
ES_α = VaR_α / (1-ξ) + (β - ξ*u) / (1-ξ),  ξ < 1

Hill 估计量 (尾指数):
ξ_Hill = (1/k) * Σ_{i=1}^{k} ln(X_{(i)}/X_{(k+1)})
其中 X_{(i)} 是降序统计量, k 是尾部观测数
```

### 3.2 Copula 引擎 (copula_engine.py)

**问题**: 当前压力测试用 `loss = MV * shock * beta` 线性缩放。多资产尾部同时崩盘（尾部依赖）完全未被建模。

**精算方案**: Copula 连接函数族 — Clayton (下尾依赖), Gumbel (上尾依赖), t-Copula (对称厚尾依赖)。

**数学基础**:

```
Sklar 定理:
对于任何联合分布 F(x₁,...,x_d), 存在一个 Copula C 使得:
F(x₁,...,x_d) = C(F₁(x₁), ..., F_d(x_d))
其中 F_i 是边缘分布

Clayton Copula (下尾依赖):
C_θ(u,v) = (u^{-θ} + v^{-θ} - 1)^{-1/θ},  θ > 0
下尾依赖系数: λ_L = 2^{-1/θ}

Gumbel Copula (上尾依赖):
C_θ(u,v) = exp(-[(-ln u)^θ + (-ln v)^θ]^{1/θ}),  θ ≥ 1
上尾依赖系数: λ_U = 2 - 2^{1/θ}

t-Copula (对称厚尾):
C_{ν,P}(u,v) = t_{ν,P}(t_ν^{-1}(u), t_ν^{-1}(v))
其中 t_ν 是自由度为 ν 的 t 分布, P 是相关矩阵

尾部依赖模拟:
1. 用 pseudo-observations 估计边缘分布 (经验 CDF)
2. 用 MLE 拟合 Copula 参数
3. 从 Copula 中采样生成联合情景
4. 计算投资组合尾部损失分布
```

**接口契约**:

### 3.3 破产理论引擎 (ruin_engine.py)

**问题**: 当前仓位限制是固定的 10%/30%/95%，不考虑波动率、胜率、或可接受的破产概率。

**精算方案**: Cramér-Lundberg 模型的离散时间近似 + Lundberg 不等式。

**数学基础**:

```
经典 Cramér-Lundberg 模型 (连续时间):
R(t) = u + ct - Σ_{i=1}^{N(t)} Y_i

其中:
  u     = 初始资本
  c     = 保费率 (交易中 = 预期收益/时间)
  N(t)  = 泊松过程 (损失事件计数)
  Y_i   = i.i.d. 损失额

破产概率:
ψ(u) = P(∃ t > 0: R(t) < 0)

Lundberg 不等式 (上界):
ψ(u) ≤ exp(-R·u)
其中 R = 调节系数, 满足 E[exp(R·Y - R·c·T)] = 1

离散时间近似 (用于量化交易):
W_n = W₀ · Π_{i=1}^{n} (1 + r_i)

其中 r_i 是每笔交易的回报

破产概率 (离散时间):
ψ(W₀) = P(min_{1≤k≤n} W_k < W_critical)

动态仓位上限:
给定可接受破产概率 ε, 最优仓位为:
f* = argmax_f {E[terminal_wealth(f)] : ψ(W₀) ≤ ε}

通过蒙特卡洛模拟估计 ψ(W₀) 并求解 f*
```

### 3.4 信度理论引擎 (credibility.py)

**问题**: 当前三路信号融合用 IC 驱动线性加权 `w_i ∝ max(IC_i, 0)`，忽略了信号源的时序稳定性。

**精算方案**: Bühlmann-Straub 信度模型。

**数学基础**:

```
Bühlmann 模型 (无风险暴露差异):
给定信号源 i 的历史表现 X_{i,1}, ..., X_{i,n}:

μ = E[X_{i,t}]           # 总体均值 (先验)
m = E[Var(X_{i,t}|Θ_i)]  # 过程方差 (within-source)
v = Var(E[X_{i,t}|Θ_i])  # 假设均值方差 (between-source)

信度因子: Z_i = n / (n + m/v)
后验估计: ˆμ_i = Z_i · X̄_i + (1-Z_i) · μ

Bühlmann-Straub 模型 (含风险暴露差异):
每个信号源的观测权重 w_{i,t} (如基于市场成交额):

信度因子: Z_i = Σw_{i,t} / (Σw_{i,t} + m/v)
后验估计: ˆμ_i = Z_i · X̄_i^w + (1-Z_i) · μ
其中 X̄_i^w = Σ(w_{i,t}/Σw_{i,t}) · X_{i,t} 是加权平均

在灵枢中的应用:
- 信号源 i ∈ {yinzi, tushenjing, zhinengti}
- X_{i,t} = 信号源 i 在 t 时刻的 IC/Rank IC
- w_{i,t} = 覆盖股票数 × 平均成交额 (市场参与度)
- ˆμ_i = 每个信号源的信度加权期望 IC
- 融合权重: α_i = softmax(ˆμ_i / σ_i) 其中 σ_i 是后验方差
```

### 3.5 Solvency II SCR 计算器 (solvency.py)

**问题**: 当前无系统化的风险资本框架。VaR 只是单一数字，没有风险分解。

**精算方案**: 简化版 Solvency II 标准公式应用于交易账户。

**数学基础**:

```
SCR (Solvency Capital Requirement) — 99.5% VaR, 1 年期限:

SCR_total = sqrt(
    SCR_market² + SCR_credit² + SCR_operational²
    + 2ρ₁₂·SCR_market·SCR_credit
    + 2ρ₁₃·SCR_market·SCR_operational
    + 2ρ₂₃·SCR_credit·SCR_operational
)

其中:
  SCR_market = max(EVT_VaR_99.5, 3·σ_daily·sqrt(252))
    — 利率, 权益, 地产, 价差, 集中度, 货币的子模块聚合
  SCR_credit  = spread_risk + default_risk
  SCR_operational = 0.03 * AUM (简化)

风险模块相关性矩阵 (EIOPA 标准):
  ρ_market,equity = 1.0
  ρ_market,credit = 0.25
  ρ_market,operational = 0.0

风险预算分配:
  Risk_Budget_i = SCR_i / SCR_total  (每个子模块的风险占比)
  组合各个仓位约束 ∝ 风险预算
```

### 3.6 VaR 回测套件 (var_backtest.py)

**问题**: 当前计算 VaR 但从不检验其准确性——无法知道模型是否在系统性低估风险。

**精算方案**: 完整的三层 VaR 回测。

**数学基础**:

```
Violation (例外):
I_t = 1 if Loss_t > VaR_t else 0

Kupiec LR 检验 (无条件覆盖):
H₀: E[I_t] = α (VaR 置信水平)
LR_uc = -2ln[(1-α)^{N-x}·α^x / (1-x/N)^{N-x}·(x/N)^x] ~ χ²(1)
其中 N = 总观测, x = 例外数

Christoffersen 检验 (条件覆盖):
检验 H₀: 例外是独立的
LR_ind = -2ln[L(p|I) / L(p₀₁,p₁₁|I)] ~ χ²(1)
联合检验: LR_cc = LR_uc + LR_ind ~ χ²(2)

Acerbi-Szekely 检验 (Expected Shortfall):
Z = (1/(N·α)) · Σ_{t=1}^{N} (Loss_t · I_t / ES_t) - 1
H₀: Z = 0 (模型正确)
通过 Bootstrap 估计 Z 的分布

Basel 红绿灯:
  Green:  1-4 例外  (无可疑)
  Yellow: 5-9 例外  (模型需审查, 乘数增加)
  Red:    ≥10 例外  (模型拒绝)
```

**接口契约**:

### 4.1 Black-Litterman 优化器 (portfolio_optimizer.py 重写)

**问题**: 当前"BL"是 `(1-c)*prior + c*view` 的标量插值，接受 `risk_aversion` 和 `tau` 参数但从不使用。

**正确方案**:

**数学基础**:

```
Step 1 — 均衡收益 (Implied Equilibrium Returns):
Π = δ · Σ · w_mkt

其中:
  δ     = 风险厌恶系数 (通常 δ = (E[r_m] - r_f) / σ_m²)
  Σ     = 协方差矩阵 (Ledoit-Wolf 收缩估计)
  w_mkt = 市值加权基准权重 (如沪深300权重)

Step 2 — 观点矩阵 (Views):
P · E(R) = Q + ε,  ε ~ N(0, Ω)

P: K×N 选择矩阵, 每行代表一个观点
Q: K×1 观点收益向量
Ω: K×K 观点不确定性对角矩阵
   Ω_ii = (P_i · Σ · P_i') · τ  (默认: 与先验成比例的不确定性)

Step 3 — 贝叶斯更新 (Black-Litterman 主公式):
E(R|views) = [(τΣ)⁻¹ + P'Ω⁻¹P]⁻¹ · [(τΣ)⁻¹Π + P'Ω⁻¹Q]
Cov(R|views) = Σ + [(τΣ)⁻¹ + P'Ω⁻¹P]⁻¹

Step 4 — 均值-方差优化:
max_w  w' · E(R|views) - (δ/2) · w' · Cov(R|views) · w
s.t.   Σw_i = 1,  w_i ≥ 0,  w_i ≤ w_max
       w' · Cov · w ≤ σ_target²  (波动率约束)
       turnover ≤ turnover_max
       sector_weights 符合限制
```

**Ledoit-Wolf 收缩估计**:
```
Σ_LW = (1-ρ) · S + ρ · μ·I

其中:
  S  = 样本协方差矩阵
  μ  = 平均方差
  I  = 单位矩阵
  ρ  = 最优收缩强度 (通过最小化 Frobenius 损失解析解确定)
```

**接口契约**:


### 4.2 分层风险平价 (hrp.py)

**问题**: 当前等权重分配是组合构建中最原始的方法。

**方案**: Marcos López de Prado 的分层风险平价算法。

**算法流程**:
```
1. 计算收益相关矩阵 ρ
2. 转换: D(i,j) = sqrt(0.5 * (1 - ρ(i,j)))  (距离矩阵)
3. 对 D 进行层次聚类 (single-linkage 或 ward)
4. 准对角化 (重排协方差矩阵按聚类顺序)
5. 递归二等分: 对每个聚类内部
   - 计算逆方差权重: w_i ∝ 1/var(cluster_i)
   - 子聚类继承权重并递归分配
```

### 4.3 市场冲击模型 (market_impact.py)

**问题**: 回测引擎使用固定 0.1% 滑点，不随订单规模变化。

**方案**: Almgren-Chriss 框架。

**数学基础**:
```
Almgren-Chriss 模型:

永久冲击 (信息泄漏):
g(v) = γ · σ · v / V

瞬时冲击 (流动性消耗):
h(v) = η · σ · (v / V)^(3/5)

总冲击成本:
C(v) = g(v) + h(v)

其中:
  v = 订单量 (股数)
  V = 日均成交量
  σ = 日波动率
  γ = 永久冲击系数 (~0.1)
  η = 瞬时冲击系数 (~0.15)

平方根模型 (简化):
impact(bps) = κ · sqrt(v / V)

其中 κ ~ 10-50 bps (依赖流动性)
```

### 4.4 归因分析 (attribution.py)

**问题**: `huice/attribution.py` 被文档引用但从未创建。

**方案**: 三维归因体系。

```
Brinson 归因:
  Allocation Effect = (w_p,i - w_b,i) · (r_b,i - r_b)
  Selection Effect   = w_b,i · (r_p,i - r_b,i)
  Interaction Effect = (w_p,i - w_b,i) · (r_p,i - r_b,i)

因子归因:
  R_p = Σ β_k · F_k + α
  归因于每个因子 k: β_k · F_k
  残差 = α (= 经理技能)

风险归因:
  边际 VaR (MVaR_i) = ∂VaR/∂w_i
  成分 VaR (CVaR_i) = w_i · MVaR_i
  Σ CVaR_i = VaR_total (加性分解)
```

### 5.2 数据窥探防御 (data_snooping.py)

**数学基础**:

```
Deflated Sharpe Ratio (DSR, Harvey & Liu 2014):

在 N 次独立试验中, 观察到的最大 Sharpe Ratio SR_max 的分布:
P(max SR > SR_obs) = 1 - Prob[SR_1 ≤ SR_obs, ..., SR_N ≤ SR_obs]

在正态假设下:
DSR = P(SR > SR_obs) = 1 - Φ( (SR_obs - E[SR]) / σ_SR )

Haircut Sharpe Ratio:
SR_haircut = SR_obs - E[max SR | N trials]
其中 E[max SR | N] ≈ σ · sqrt(2·log(N))

Probabilistic Sharpe Ratio (PSR, Bailey & López de Prado):
PSR(SR*) = Prob[SR > SR*]
       = Φ( (SR_obs - SR*) · sqrt(T-1) / sqrt(1 - γ₃·SR_obs + (γ₄-1)/4·SR_obs²) )

其中 SR* = 基准 Sharpe (如 0), T = 观测数, γ₃ = 偏度, γ₄ = 峰度
```

### 5.3 GARCH 引擎 (garch_models.py)

**数学基础**:

```
GARCH(1,1):
r_t = μ + ε_t,  ε_t = σ_t · z_t,  z_t ~ i.i.d. N(0,1) 或 t(ν)
σ_t² = ω + α · ε_{t-1}² + β · σ_{t-1}²
约束: ω > 0, α ≥ 0, β ≥ 0, α+β < 1 (平稳)

EGARCH(1,1) — 捕捉杠杆效应:
ln(σ_t²) = ω + α·(|z_{t-1}| - E[|z_{t-1}|]) + γ·z_{t-1} + β·ln(σ_{t-1}²)
γ < 0 → 坏消息 (负冲击) 增加波动率更多

GJR-GARCH(1,1) — 非对称 GARCH:
σ_t² = ω + α·ε_{t-1}² + γ·I(ε_{t-1}<0)·ε_{t-1}² + β·σ_{t-1}²
γ > 0 → 负冲击有额外影响

条件 VaR (基于 GARCH):
VaR_t(α) = μ + σ_t · q_α(z)

其中 q_α(z) 是标准化残差分布的 α 分位数.
如果用 t 分布: q_α(z) = t_ν^{-1}(α) · sqrt((ν-2)/ν)
如果用正态分布: q_α(z) = Φ^{-1}(α)
```

### 5.5 Walk-Forward 交叉验证 (cross_validation.py)

**数学基础**:

```
Purged K-Fold (时间序列):

传统 K-Fold 对时间序列无效 → 信息泄漏.

Purged K-Fold:
1. 将数据分为 K 个连续的时间段

Walk-Forward:
```

```python
from dataclasses import dataclass
from pathlib import Path
from decimal import Decimal

@dataclass
class AppConfig:
    version: str
    data: DataConfig
    factor: FactorConfig
    model: ModelConfig
    fusion: FusionConfig
    backtest: BacktestConfig
    risk: RiskConfig
    api: APIConfig
    logging: LoggingConfig

class ConfigLoader:
    _instance = None
    _config = None

    @classmethod
    def load(cls, path: str = "config.yaml") -> AppConfig:
        """单例加载, 环境变量模板替换"""
        if cls._config is None:
            raw = yaml.safe_load(Path(path).read_text())
            cls._config = cls._parse(raw)
        return cls._config
```

## 11. 验证方法论

### 11.1 数学正确性验证

每个精算引擎通过对已知分布的合成数据进行测试来验证：

| 引擎 | 合成数据 | 成功准则 |
|------|---------|---------|
| EVTEngine | 从 GPD(ξ,β) 生成, u 已知 | 恢复 ξ ± 0.05, β ± 0.10 |
| CopulaEngine | 从 Clayton(θ=2) 生成 | 恢复 θ ± 0.15, λ_L ± 0.05 |
| RuinEngine | 已知破产概率的对称随机游走 | 估计 ψ 在真值 ± 0.02 内 |
| CredibilityEngine | 3 源, 已知过程/假设方差 | 后验权重在真实最优的 5% 内 |
| VaRBacktest | 已知违约率 (恰好 α) 的序列 | Kupiec p > 0.30 (不应拒绝) |

### 11.2 集成验证

1. **全量测试**: `python -m pytest tests/ -v` — 目标 ≥ 600 passed
2. **精度扫描**: `grep -r "float" juece/ yinzi/ jingsuan/` — 零漏网
3. **VaR 回测**: 在 5 年历史数据上, EVT VaR 的 Kupiec p > 0.10 (正态 VaR 预期 p < 0.01)
4. **Walk-Forward 回测**: 全时段 vs 买入持有的夏普比率比较
5. **归因完整性**: Σ 归因效应 = 总主动回报 (误差 < 1bp)
