## 一、项目概述

### 1.1 为什么叫"灵枢"

灵枢是中国古代医学经典《黄帝内经》的一部分，讲的是人体经络系统中"灵动枢纽"的运作机理。类比到股票市场：五千多只A股通过产业链、资金链、信息链交织成一个**有机生命体**，本系统如同一套**经络诊断系统**——通过多维度因子（脉搏）感知市场状态，通过AI智能体（思辨）分析信息本质，通过图神经网络（经络）捕捉个股间的联动传导，最终精准定位价值的"穴位"。

### 1.2 项目背景与核心问题

中国A股市场超过5000只上市股票，每日产生的结构化数据（行情、财务）和非结构化数据（新闻、公告、社交媒体）规模巨大。传统量化策略面临三个核心困境：

| 困境 | 传统做法 | 痛点 |
|------|----------|------|
| **信息处理瓶颈** | 人工阅读研报/新闻 | 一个人一天只能深度覆盖10-20只股票 |
| **静态模型失效** | 固定权重多因子 | 市场风格切换（价值↔成长↔主题）时模型失速 |
| **孤立分析盲区** | 个股独立打分 | 忽视产业链传导，错过"宁德时代涨→锂矿涨→电解液涨"的扩散效应 |

**本系统的核心命题：能不能构建一个AI系统，它像一支由多个专业分析师组成的团队，各自负责不同维度的分析，通过结构化的协作机制，持续产出超越人类覆盖能力的投资洞见？**

### 1.3 核心技术壁垒

## 二、核心技术原理

### 2.1 LLM多智能体协作框架（核心创新）

这是本系统最大的创新点。我们构建了5个专业化的LLM Agent，每个Agent拥有独立的分析视角和工具集：

#### 智能体通信协议

```python
# 每个Agent输出的结构化数据格式
@dataclass
class AgentOutput:
    agent_id: str           # 智能体标识
    timestamp: datetime     # 分析时间
    target_stocks: List[str]  # 分析覆盖的股票
    signal: float           # [-1, 1] 看空到看多信号
    confidence: float       # [0, 1] 置信度
    reasoning: str          # 可读的推理过程（关键：前端展示用）
    evidence: List[Dict]    # 支撑证据（数据来源+具体数值）
    risk_flags: List[str]   # 识别的风险点
```

#### 为什么这是真正的壁垒

1. **提示词工程复杂性**：每个Agent需要精心设计的系统提示词，包含金融领域知识、分析框架、输出格式约束
2. **工具链集成**：Agent需要调用外部API（新闻、财报、行情），这需要稳定的RAG管道和函数调用链
3. **幻觉防控**：金融场景对准确性要求极高，需要多层次的事实核查机制
4. **成本优化**：5个Agent并行运行需要合理的模型选择和缓存策略
5. **中文金融NLP**：中文财报语言的特殊性（"业绩符合预期"≠利好，"超预期但低于市场传闻"≠利好）

### 2.2 图神经网络 — 产业链关系建模

#### 为什么需要GNN

传统多因子模型将每只股票视为独立样本，但实际上A股存在强烈的**同涨同跌**和**产业链扩散**效应。例如：

```
宁德时代发布超预期财报
  → 直接相关: 锂电池产业链 (赣锋锂业、天赐材料、恩捷股份)
  → 间接相关: 新能源汽车 (比亚迪、理想汽车)
  → 更远传导: 锂矿 (天齐锂业) → 碳酸锂期货 → 盐湖提锂概念
```

这种传导关系用传统协方差矩阵无法捕捉，但用图神经网络可以自然地建模。

#### GNN建模方案

```
图构建:
  节点 = A股所有股票 (5000+)
  边类型:
    ├── 产业链上下游关系 (从财报/供应链数据提取)
    ├── 同行业竞争/合作关系 (申万行业分类)
    ├── 机构共同持仓关系 (基金重仓股交叉)
    └── 概念板块归属关系 (Wind/同花顺概念)

模型架构:
  输入: 股票特征向量 (因子值 + Agent评分)
    ↓
  图卷积层1: 产业链信息聚合 (按边类型分别卷积)
    ↓
  图卷积层2: 跨行业信息传播
    ↓
  图注意力层: 学习不同边类型的重要性权重
    ↓
  输出: 每只股票的"信息增强"因子得分 (融合了关联股票信息)
```

**为什么GNN构成壁垒：**
- 高质量的产业链关系图需要大量数据清洗工作（不是公开数据集能直接用的）
- 异构图的多种边类型设计需要领域知识
- 训练和推理的计算复杂度远高于传统模型
- 图结构的动态更新（产业链随时间变化）增加了系统复杂度

### 2.3 时变多因子模型（量化基础层）

保留并增强传统多因子体系作为系统的"底盘"：

#### 因子体系（5大类 25+因子）

| 类别 | 因子 | 计算方式 | 更新频率 |
|------|------|----------|:--:|
| **估值** | PE、PB、PS、FCF Yield、PEG | 财报+行情 | 日频 |
| **动量** | 1月/3月/6月/12-1月动量、换手率动量 | 行情计算 | 日频 |
| **质量** | ROE、ROA、毛利率、净利率、现金流/营收 | 财报 | 季频→日频插值 |
| **波动率** | 历史波动率、下行波动率、Beta、VaR | 行情计算 | 日频 |
| **情绪** | 成交量比、资金流向、换手率异常、北向资金 | 行情+资金 | 日频 |
| **另类** | 分析师覆盖变化、机构持股变化、股东人数变化 | 另类数据 | 周频 |
| **AI** | Agent情绪评分、Agent基本面评分、GNN聚合得分 | 系统产出 | 日频 |

#### 时变权重（卡尔曼滤波）

传统做法使用固定权重或滚动回归，本项目使用卡尔曼滤波实现因子权重的在线自适应估计：

```
状态空间模型:
  观测方程: r_t = F_t · β_t + ε_t    (收益 = 因子暴露 × 时变权重 + 噪声)
  状态方程: β_t = β_{t-1} + η_t      (权重遵循随机游走)

  其中 ε_t ~ N(0, R),  η_t ~ N(0, Q)
  β_t 通过卡尔曼滤波的预测-更新两步迭代实时估计
```

**创新点：** 将AI Agent的结构化输出作为额外"因子"接入卡尔曼滤波框架，实现传统量化 + AI的有机融合。

### 2.4 集成决策引擎

三层模型产出的信号通过集成决策引擎融合：

```
最终得分 = w1 × 多因子得分  +  w2 × GNN增强得分  +  w3 × Agent综合评分

其中:
  - 多因子得分 = Σ(卡尔曼权重_j × 因子暴露_j)
  - GNN增强得分 = GNN(因子向量, 产业链图)  [融合了关联股票信息]
  - Agent综合评分 = 0.25×宏观 + 0.25×赛道 + 0.30×个股 + 0.20×情绪
  - w1, w2, w3 由历史IC表现动态调整
```

### 2.5 风险控制体系

```
风控层次:
  L1: 熔断器 (Circuit Breaker)
      CLOSED ↔ OPEN ↔ HALF_OPEN (三态自动切换)
      触发条件: 连续亏损≥3次 | 日亏损≥2% | 最大回撤≥8%

  L2: 仓位管理
      单票 ≤ 10% | 单行业 ≤ 30% | 总仓位 ≤ 95%
      凯利公式动态调整每笔交易的仓位大小

  L3: 实时风险监控
      VaR(95%) / CVaR(95%) 实时计算
      压力测试: 2008/2015/2020/2024 历史极端场景回放
      黑天鹅检测: 舆情Agent异常信号 → 自动降仓

  L4: 智能体风险Agent
      独立运行，有权否决交易（与收益导向的Agent形成制衡）
```

### 3.2 核心数据流

```
每日15:00收盘后触发:
  1. 数据层拉取: 全市场日线 + 财报更新 + 新闻舆情
  2. 因子引擎: 25+因子并行计算
  3. AI智能体: 5 Agent并行分析，产出结构化洞见
  4. GNN引擎: 产业链图更新 + 信息传播
  5. 卡尔曼滤波: 基于最新收益更新因子权重
  6. 集成决策: 三路信号加权融合 → 全市场排序
  7. 调仓计算: Top 30 选股 + 风险过滤 + 交易清单
  8. 模拟执行: 生成成交记录
  9. 持久化: 因子/信号/交易/智能体报告写入数据库
  10. WebSocket推送: 前端Dashboard实时更新

实时模式（交易时段 9:30-15:00）:
  - 每5分钟拉取快照数据
  - 舆情Agent持续监控异常信号
  - 风险Agent实时评估持仓风险
  - 前端每10秒推送市场概览更新
```
