## 一、环境激活

每次打开终端后，先进入项目并激活虚拟环境：

```bash
cd E:\28721\lingshu
source .venv/Scripts/activate
```

激活成功后终端前会显示 `(.venv)`，此时 `python` = Python 3.11.9。

```bash
# 验证
python --version          # Python 3.11.9
pip list | grep lingshu   # lingshu 2.0.0 (editable)
```

## 二、运行测试

```bash
# 运行全部测试
python -m pytest tests/ -v

# 运行单模块测试
python -m pytest tests/test_yinzi/ -v
python -m pytest tests/test_shuju/ -v
python -m pytest tests/test_tushenjing/ -v
python -m pytest tests/test_juece/ -v
python -m pytest tests/test_fengkong/ -v
python -m pytest tests/test_zhixing/ -v
python -m pytest tests/test_huice/ -v
python -m pytest tests/test_shujuku/ -v

# 跳过慢速/集成测试
python -m pytest tests/ -v --ignore=tests/test_jiekou --ignore=tests/test_integration_real.py

# 覆盖率报告
python -m pytest tests/ --cov=. --cov-report=html
```

## 三、启动 API 服务

```bash
# 开发模式（自动重载）
uvicorn jiekou.server:app --host 0.0.0.0 --port 8000 --reload

# 生产模式
uvicorn jiekou.server:app --host 0.0.0.0 --port 8000

# 自定义 LLM 后端
LLM_BACKEND=deepseek LLM_API_KEY=xxx uvicorn jiekou.server:app --port 8000
```

启动后访问：
- API 文档: http://localhost:8000/docs
- 健康检查: http://localhost:8000/api/health

## 五、模块导入

安装 `pip install -e .` 后，所有模块可作为顶层包直接导入：

```python
# 任何位置都可以这样导入（不需要 sys.path.insert）
from yinzi.factor_base import FactorBase
from juece.ensemble_engine import EnsembleEngine
from shuju.data_preprocessor import DataPreprocessor
from zhinengti.agent_base import AgentBase
from tushenjing.gnn_model import GCNModel
from fengkong.risk_manager import RiskManager
from zhixing.order_manager import OrderManager
from huice.backtest_engine import BacktestEngine
from shujuku.session import get_session
from jiekou.server import app
```

## 六、依赖管理

```bash
# 查看已安装版本
pip list

# 安装新依赖后更新锁文件
pip freeze > requirements-lock.txt

# 重新安装（如果依赖出问题）
pip install -e ".[dev]" --force-reinstall

# 只装核心依赖（不含开发/测试）
pip install -e .
```

## 七、环境重置

如果虚拟环境损坏，可完全重建：

```bash

source .venv/Scripts/activate

pip install -e ".[dev]"
```
